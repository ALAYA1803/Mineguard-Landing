Technology

HOW THE MESH WORKS

01

Signal Detection

IoT nodes mounted on vehicles and infrastructure continuously broadcast GNSS positions via V2X protocol. The mesh self-organizes to maintain coverage even in deep pit zones.

02

Edge Processing

The onboard Edge Node analyzes proximity data locally. Collision trajectories are computed using predictive algorithms — no cloud dependency for critical alerts.

03

Multi-Layer Alert

At 43ms, simultaneous alerts fire: to the light vehicle HMI device, the autonomous truck's FMS, and the control room supervisor dashboard — all before a human can blink.

04

Safe Resolution

Geofenced corridors auto-update. The incident is logged with GPS coordinates, timestamps and vehicle IDs — feeding the analytics dashboard for continuous safety improvement.

FAQ

COMMON QUESTIONS

How does MineGuard integrate with existing autonomous fleets?

MineGuard uses industry-standard V2X communication protocols to connect with Fleet Management Systems (FMS). It is installed non-intrusively as an additional safety layer — no modifications to existing vehicle firmware are required.

What is the precision level of the positioning system?

MineGuard utilizes high-precision GNSS with RTK corrections, providing centimeter-level accuracy (±2cm). This precision is critical for safe interaction between light vehicles and autonomous mining giants weighing over 300 tonnes.

Does it work in areas without conventional internet coverage?

Yes. MineGuard operates on private LTE/5G mining networks and has a local Mesh Network fallback mode. Critical proximity alerts are issued even with zero central server connectivity.

What is the standard implementation timeframe?

A standard deployment for a fleet of 20 light vehicles takes approximately 2 weeks, including digital environment calibration, IoT node positioning, FMS integration, and full personnel training.
